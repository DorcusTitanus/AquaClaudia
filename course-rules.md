# MULTI 211: Adobe Photoshop II — Course Rules

Governs LLM-assisted development for MULTI 211. Inherits from `global-rules.md`;
this file wins on conflict.

## Course at a Glance
- 13-week fully online Adobe Photoshop II course
- Gamified "Level" structure over Horton's Absorb → Do → Connect pedagogy
- Core themes: non-destructive editing, layer-based workflows, compositing
  logic, process transparency, peer critique, reflection

## Structural Invariants (never alter without explicit approval)

**Level template — exactly 6 parts, in this order:**
1. Overview page (LLOs, time estimate, badge, XP)
2. Absorb content (tutorial)
3. Absorb Check (auto-graded quiz, unlimited attempts)
4. Project (PSD submission)
5. Peer review (Canvas-native)
6. Weekly journal (private reflection)

**Weekly rhythm:** Fri unlock → Tue Absorb Check → Thu Project → Sun Peer + Journal

**Per-level points:** 40 pts standard — Tutorial 5 + Checkpoint 5 + Project 25
+ Journal 5. Some levels with extra steps may total 50 pts; flexibility is OK
when the level's content justifies it. XP is conceptual; points are for grade
transparency. Do not invent alternate grading systems.

Project is graded via the Peer Review rubric (5 criteria × 5 pts binary
Meets/Not Yet, hide_points). Tutorial and Journal use reusable rubrics;
Checkpoint is a quiz with min_score 4.0 to pass.

**Level 1 is locked:** "Level 1: Mask Architect" — non-destructive masking,
selections, layer masks, refinement, reuse, blending modes, layer organization.

## Writing Rules (student-facing content)
- 8th-grade reading level. Short sentences. No jargon.
- No meta commentary, no breaking the game narrative, no instructor-facing asides.
- No emojis unless Jefe explicitly asks.
- Calm, clear, supportive, confident, human. Not apologetic or over-explaining.

## Pedagogy Alignment
All outputs must reinforce non-destructive workflows, process over polish, and
PSD inspection during peer review. Avoid tool trivia, surface-level tips,
redundant content, and scope creep.

**Cognitive load:** Never introduce advanced techniques before foundations.
Level 1 stays at masking + blending modes — no displacement maps, surface
realism, or advanced distortion unless explicitly instructed.

## Naming Conventions
- Levels: `Level X: Skill Identity`
- Projects: `Project: Skill Name Composite`
- Journals: `Journal: Level X Reflection`
- Never collapse level identity into assignment names. Never use em dashes
  to merge container + task.

## Cross-Reference Discipline
Levels get renamed, reordered, remixed. Hard cross-references create update debt.

Priority order:
1. **No cross-reference** — preferred. Omit if not essential.
2. **Generic** — "the next level," "a later level," "the previous level."
3. **Specific** — only when a student genuinely needs the exact name to navigate.

Never hard-reference a future level's name or number in current content unless
structurally required (e.g., badge grid position).

## Gamification Boundaries
Badges represent demonstrated skill, not participation. Don't change point
values, XP logic, or grading systems.

## Accessibility (always on)
- Semantic heading structure (H2–H4; page title ≠ first H2)
- Alt text under 120 characters, no "image of" or file extensions
- No instructions that depend on visuals alone
- Canvas-friendly HTML
- WCAG 2.1 AA is the floor. Full rubric in `docs/multi211/accessibility-cvc-oei.md`.

## AI Policy (for student-facing content)
- Journals: explicitly prohibit AI generation, framed as purpose-driven
- Elsewhere: AI may be acknowledged as a tool

## Output Discipline
- HTML must be Canvas-ready
- Wiki pages in IMSCC cartridges must be full HTML documents with the
  required `<html><head><body>` structure and meta headers — bare fragments
  will not import correctly
- IMSCC cartridges must import as unpublished/draft — never auto-publish
- Quizzes follow Canvas QTI standards
