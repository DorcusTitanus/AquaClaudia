## PART 1: COURSE IDENTITY AND STRUCTURE

### Course Identity

MULTI 211 is a 13-week, fully online Adobe Photoshop II course built around:

- Non-destructive editing
- Layer-based workflows
- Professional compositing logic
- Process transparency
- Peer critique
- Reflection

The course uses a gamified "Level" structure. The pedagogy underneath follows Horton's Absorb → Do → Connect model.

### Course Structure

**Pre-Levels**

- Orientation
- Getting the Software
- Prologue (game logic + pretest + mastery paths)

**Levels 1–12**

- One level per week
- Spring Break: no work

Each Level contains exactly:

1. Overview page (LLOs, time estimate, badge, XP)
2. Absorb content (tutorial)
3. Absorb Check (auto-graded quiz, unlimited attempts)
4. Project (PSD submission)
5. Peer review (Canvas-native)
6. Weekly journal (private reflection)

### Weekly Rhythm

- Friday → Level unlocks
- Tuesday → Absorb Check due
- Thursday → Project submission due
- Sunday → Peer review + journal due

Students succeed by checking in consistently (M/W/F).

### Assessment Weighting (Per Level)

- 20% Absorb
- 50% Project
- 20% Peer Review
- 10% Journal

Total: 50 points per level.

XP exists conceptually. Points are for grade transparency.

### Level 1 Identity (Locked)

- **Level Name:** Mask Architect
- **Core Skill:** Non-destructive masking
- **Focus:** Selections, Layer masks, Mask refinement, Mask reuse, Blending modes, Layer organization

## PART 2: LLM OPERATING RULES

Follow these rules strictly when generating any content for MULTI 211.

### 1. Structural Integrity

- Never alter the 6-part level template.
- Never introduce new structural elements without explicit approval.
- Do not suggest sequential lock unless requested.
- Assume Prerequisites + Complete All Items is the gating system.

### 2. Pedagogy Alignment

All outputs must:

- Align to Horton's Absorb–Do–Connect
- Reinforce non-destructive workflows
- Emphasize process over polish
- Support PSD inspection during peer review

Avoid:

- Tool trivia
- Surface-level Photoshop tips
- Redundant content
- Scope creep

### 3. Writing Style Constraints

All student-facing content must:

- Be written at approximately 8th-grade reading level
- Use short sentences
- Avoid academic jargon
- Avoid instructor-facing commentary
- Avoid breaking the game narrative

Do not:

- Insert meta commentary
- Reference internal design logic
- Use unnecessary emojis
- Add fluff

### 4. Naming Conventions

Levels:

- Format: `Level X: Skill Identity`

Assignments:

- Format: `Project: Skill Name Composite`
- Format: `Journal: Level X Reflection`

Never:

- Collapse level identity into assignment names
- Use em dashes to merge container + task

### 5. Accessibility Compliance

Always:

- Use semantic heading structure
- Provide alt text under 120 characters
- Avoid image-dependent instructions
- Keep HTML Canvas-friendly

### 6. Gamification Boundaries

XP is conceptual. Points are structural.

Do not:

- Change point values
- Alter XP logic
- Suggest alternate grading systems

Badges represent demonstrated skill, not participation.

### 7. AI Policy Reinforcement

In journals:

- Explicitly prohibit AI generation
- Frame prohibition as purpose-driven

Elsewhere:

- AI may be acknowledged as a tool

### 8. Cognitive Load Control

Never:

- Introduce advanced techniques before foundational ones are internalized
- Add complexity to Level 1 beyond masking + blending modes

Save for later levels unless explicitly instructed:

- Displacement maps
- Surface realism
- Advanced distortion

### 9. Tone

Tone must be:

- Calm
- Clear
- Supportive
- Confident
- Human

Avoid:

- Apologetic tone
- Over-explaining
- Over-engineering

### 10. Output Discipline

When generating:

- HTML must be Canvas-ready
- Quizzes must follow Canvas QTI standards
- Keep responses structured
- Do not add extra commentary unless requested

### 11. Cross-Reference Discipline

Levels change names, shift positions, and get remixed. Hard cross-references between levels create cascading update debt.

Priority order:

1. **No cross-reference** — Preferred. If the information is not essential, omit it.
2. **Generic cross-reference** — When a reference is necessary, keep it generic (e.g., "the next level," "a later level," "the previous level").
3. **Specific cross-reference** — Only when a student genuinely needs the exact name to navigate or complete a task (rare).

Apply this to:

- Overview pages ("Advance to the next level" — not "Advance to Level 5: Production Designer")
- Level Unlocked pages ("You are now ready to begin the next level" — not the next level's name)
- Tutorial text ("In a later level, you will learn…" — not "In Level 9…")
- Badge Collection pages (badge names within the grid are the canonical identity, not cross-references — these are fine)

Never:

- Hard-reference a future level's name in a current level's content
- Hard-reference a future level's number unless it is structurally required (e.g., badge grid position)

## PART 3: PROJECT RULES & STANDARDS

Source: MULTI_211_Project_Rules.pdf (Authoritative Project Reference)

### Course Scope

- 13-week asynchronous online course
- Prerequisite: MULTI 111
- Primary tool: Adobe Photoshop
- Focus on non-destructive professional workflows

### Pedagogical Framework

- Hidden Horton Absorb–Do–Connect model
- 6 instructional levels plus final portfolio week
- Emphasis on revision, collaboration, and authentic practice

### Gamification & Points

- Each level totals 50 points
- Tutorial: 5 points
- Checkpoint quiz: 5 points
- Group project deliverable: 40 points
- All quizzes titled "Checkpoint"

### Project Rules

- All major projects are group-based
- Flexible roles supported for UDL alignment
- Each project must include purpose, steps, deliverables, and evaluation criteria

### Canvas Structure

- Modules follow: Overview, Absorb, Do, Connect, Assessment
- Semantic headings only (H2–H4)
- Page title and first H2 may not repeat

### Accessibility & Compliance

- WCAG 2.1 Level AA compliance required
- All images include alt text under 120 characters
- Videos must be captioned; audio requires transcripts
- Instructions may not rely on visuals alone

### External Links & Tools

- All external links open in new tabs
- Links must include Canvas `inline_disabled` class and color override
- Third-party tools must be accessible or optional

### File & Technical Standards

- Photoshop files are screen-based at 72 ppi
- Maximum canvas size: 1920 × 1280 pixels
- Designed for manageable upload and review

### Assessment Philosophy

- Assessment prioritizes understanding over speed
- Peer review uses structured prompts or rubrics
- Reflection activities are guided

### Badges & Visual Standards

- Locked badge: gray question mark embroidery style
- Earned badge Level 1: Mask Architect
- File naming and dimensions are locked standards

### AI Use

- AI tools may support learning but not replace decision-making
- Students must demonstrate understanding and intent

## PART 4: CVC-OEI COURSE DESIGN RUBRIC REFERENCE

Source: CVCOEICourseDesignRubricrev_10_2018.pdf (CC BY 4.0)

This rubric governs course design quality for CVC-OEI alignment. The course should target "Aligned" or "Additional Exemplary Elements" on every applicable criterion.

### Section A: Content Presentation

**A1–A3: Unit-Level Objectives**

- A1: Objectives consistently placed and easy to locate in each learning unit
- A2: Objectives include demonstrable, student-centered learning outcomes
- A3: Content aligned with unit objectives; connections made explicitly clear to students

**A4–A8: Use of CMS**

- A4: Navigation and content flow easily determined; tutorial materials for course navigation included
- A5: Content segmented into distinct units; consistently structured and sequenced to reduce cognitive load
- A6: Page content chunked with descriptive headings and subheadings
- A7: CMS tools reduce labor intensity and streamline access
- A8: Variety of media used creatively to facilitate student-centered learning

**A9–A11: Learner Support**

- A9: Instructions for working with content are directly embedded with the content
- A10: Individualized learning opportunities provided (remedial and advanced)
- A11: Opportunities for anonymous feedback during and after course delivery

**A12–A14: Institutional Support**

- A12: Policies relevant to learner success included, easy to find, and placed where most relevant
- A13: Links to institutional services included, clearly labeled, and placed where needed most
- A14: Technology support explained; links and troubleshooting tips provided throughout

### Section B: Interaction

**B1–B3: Instructor Contact**

- B1: Instructor initiates contact prior to or at beginning of course with multiple resources
- B2: Regular instructor-initiated contact using CMS communication tools
- B3: Students encouraged to initiate contact; multiple means provided; expected response times included

**B4–B6: Student-to-Student Contact**

- B4: Student-initiated interaction available and encouraged with variety of tools
- B5: Regular effective contact among students designed to build community
- B6: Guidelines for required participation levels consistently provided; rubrics included

### Section C: Assessment

**C1–C4: Effective Assessment**

- C1: Assessments designed for authenticity; mimic authentic environments to facilitate transfer
- C2: Assessments align with and match objectives; learners directed to appropriate objectives
- C3: Both formative and summative assessments; multiple types used
- C4: Frequent assessments at regular intervals with timely feedback

**C5–C8: Guidance and Feedback**

- C5: Rubrics or descriptive criteria included; models of "good work" provided
- C6: Instructions clearly explain how to complete assessments with exemplary detail
- C7: Clear description of meaningful, timely feedback; guidance on applying feedback
- C8: Multiple opportunities for self-assessment that encourage students to seek help

### Section D: Accessibility

All elements must be Aligned. Key requirements:

- D1: Semantic heading styles used in correct order
- D2: Lists created with proper list tools
- D3: Links use meaningful text (not raw URLs)
- D4: Tables have designated header cells
- D5: Sufficient color contrast (foreground/background)
- D6: Color not used as sole means of conveying information
- D7: All images have appropriate alt text (no "image of" or file extensions)
- D8: Reading order correctly set
- D9: Slides use built-in accessible layouts with unique titles
- D10: Spreadsheets include labels and textual descriptions
- D11: Files pass built-in accessibility checkers
- D12: All video has accurate captions
- D13: Audio files accompanied by complete transcripts
- D14: Flashing content only if instructionally needed; within flash thresholds
- D15: Live/synchronous video includes means for captions
- D16: No auto-play for audio/video content

## DRIFT PREVENTION

Without these rules, an LLM will:

- Drift tone
- Increase reading level
- Introduce structural inconsistency
- Overcomplicate assignments
- Break the gamification logic
- Undermine accessibility compliance

This document prevents that drift. Reference it at the start of every session.
